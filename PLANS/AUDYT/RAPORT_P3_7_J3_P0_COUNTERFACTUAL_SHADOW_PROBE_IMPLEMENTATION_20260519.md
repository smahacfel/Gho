# RAPORT P3.7-J3 P0 Counterfactual Shadow Probe Implementation

Date: 2026-05-19

Status: PARTIAL harness PASS / superseded by P0R

## Verdict

The original P3.7-J3 P0 implementation created a useful counterfactual probe
configuration surface, deterministic sampler, selection/skip logs, and a narrow
transport/entry join-key harness.

It did not fully implement the P3.7-J3 P0 runtime contract. This report is
therefore superseded by:

```text
PLANS/AUDYT/RAPORT_P3_7_J3_P0R_REPAIR_IMPLEMENTATION_20260519.md
```

## What P0 Proved

Implemented and validated at harness level:

- disabled-by-default `[p37_shadow_probe]` config surface,
- fail-closed shadow-only profile validation,
- deterministic `deterministic_hash_mod` selection,
- `probe_selected` and `probe_skipped` JSONL rows,
- basic probe join metadata:
  - `source_ab_record_id`,
  - `probe_id`,
  - `dispatch_source`,
  - `collection_plane`,
  - `probe_plane`,
- isolated probe transport/entry paths,
- basic probe-aware join-key audit fixture.

## What P0 Did Not Prove

The original P0 did not prove runtime execution feasibility for the probe plane:

- probe transport/entry rows were generated by a synthetic harness path,
- `execution_outcome=counterfactual_probe_recorded_no_simulation` was not a
  real shadow simulation result,
- the hook did not route selected probes through the real
  `ShadowSimulator::simulate_buy(...)`,
- runtime bounds were not enforced in shared runtime state,
- `append=false` / `append=true` namespace semantics were incomplete,
- the join-key audit could pass probe continuity without exact decision/V3 row
  join coverage.

## Governance

This correction does not invalidate ADR-0136. It narrows the status of the
original P0 implementation and explicitly prevents synthetic no-simulation rows
from being treated as P0 success evidence.

The next valid gate is P0R code-level repair, followed by a bounded R15 runtime
smoke.
